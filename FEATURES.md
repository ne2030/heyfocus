# HeyFocus 기능 명세

> "멘탈 스레드" 개념 기반의 집중력 관리 앱

---

## 태스크 관리

| 기능 | 설명 |
|------|------|
| **태스크 추가** | 하단 인풋 창에 텍스트 입력 후 `Enter` 또는 Add 버튼 클릭. Active가 5개 미만이면 Active에, 5개면 Later에 자동 추가 |
| **태스크 편집** | 태스크 텍스트 더블클릭으로 인라인 편집 모드 진입. `Enter` 또는 바깥 클릭으로 저장 |
| **태스크 삭제** | 태스크 우측 X 버튼 클릭, 또는 태스크 선택 상태에서 `D` 키 |
| **태스크 완료** | 태스크 우측 체크(✓) 버튼 클릭. 완료된 태스크는 목록에서 제거되고 로그에 기록 |
| **Active/Later 이동** | 드래그앤드롭으로 섹션 간 이동. 선택 상태에서 `L` 키로 Later로 이동 |

---

## 포커스 시스템

| 기능 | 설명 |
|------|------|
| **포커스 설정** | 태스크 좌측 원형 인디케이터 클릭 또는 선택 상태에서 `Space` 키. 포커스된 태스크는 주황색 테두리와 글로우 효과로 강조 |
| **슬롯 인디케이터** | 헤더에 5개 점으로 Active 태스크 현황 표시 (채워진 점=사용 중, 강조된 점=포커스된 태스크) |
| **자동 포커스 전환** | 포커스된 태스크 완료 시 다음 Active 태스크로 자동 포커스 이동 |

## 키보드 단축키

### 네비게이션

| 단축키 | 기능 |
|--------|------|
| `↑` / `↓` | Active 태스크 간 이동 (순환) |
| `Cmd+1~5` | Active 태스크 인덱스로 직접 선택 |
| `Escape` | 선택 해제 |
| `Cmd+N` | 태스크 추가 입력창에 포커스 |

### 태스크 조작

| 단축키 | 기능 |
|--------|------|
| `Space` | 선택된 태스크 포커스 토글 |
| `D` | 선택된 태스크 삭제 |
| `L` | 선택된 태스크를 Later로 이동 |
| `I` | 선택된 태스크 편집 모드 진입 |
| `Cmd+Z` | 실행취소 (Undo) |

### 윈도우 제어

| 단축키 | 기능 |
|--------|------|
| `Cmd+P` | 항상 위에 표시 토글 |
| `Cmd+M` | 컴팩트 모드 토글 |
| `Cmd+[` | 투명도 5% 감소 |
| `Cmd+]` | 투명도 5% 증가 |
| `Cmd+S` | 설정 패널 열기 |

---

## 윈도우 관리

| 기능 | 설명 |
|------|------|
| **항상 위에 표시** | 핀 버튼 또는 `Cmd+P`로 다른 창 위에 고정 |
| **컴팩트 모드** | `Cmd+M`으로 섹션 헤더, 인풋창, Later 섹션 숨김. 최소화된 UI |
| **투명도 조절** | `Cmd+[`/`Cmd+]`로 5%씩 조절 (30~100%), 또는 설정 패널 슬라이더 |
| **자동 크기 조절** | 컨텐츠에 따라 윈도우 높이 자동 조절 |

---

## 실행취소 (Undo)

| 기능 | 설명 |
|------|------|
| **Undo** | `Cmd+Z`로 마지막 작업 되돌리기 |
| **지원 작업** | 추가, 삭제, 완료, 이동, 포커스 변경, 편집 모두 지원 |

---

## 활동 로그 & 통계

| 기능 | 설명 |
|------|------|
| **설정 패널** | `Cmd+S` 또는 톱니바퀴 버튼으로 하단 시트 열기 |
| **통계** | 완료 수(Done), 포커스 전환 수(Switches), 총 이벤트 수(Events) 표시 |
| **로그 창** | "Open Log Window" 버튼으로 별도 창에서 상세 활동 기록 확인 |

### 기록되는 이벤트

- `TASK_CREATED` - 새 태스크 추가
- `MOVE_TO_ACTIVE` - Active로 이동
- `MOVE_TO_LATER` - Later로 이동
- `SWITCH_FOCUS` - 포커스 전환
- `TASK_DONE` - 태스크 완료
- `TASK_DELETED` - 태스크 삭제
- `TASK_EDITED` - 태스크 텍스트 수정

---

## 제약 조건

- Active 태스크는 최대 **5개**까지만 가능 (인지 부하 제한)
- 태스크 텍스트는 최대 **100자**
- 투명도 범위: **30% ~ 100%**
